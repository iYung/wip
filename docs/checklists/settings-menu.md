## Settings Menu Checklist

- [ ] Task 1 — `lua/game/scenes/settings_menu.lua` — Create new file. Define a plain Lua table `SettingsMenu` (not a Scene subclass). Fields: `is_open = false`, `selected = 1`, `_prev_up`, `_prev_down`, `_prev_confirm`, `_prev_escape`. Items list: `{"Fullscreen / Window", "Leave Game"}`. Implement `SettingsMenu.new()` that loads `assets/start_btn.png` and `assets/start_btn_selected.png` and stores them as `self._img_btn` / `self._img_btn_sel`. Implement `SettingsMenu:open()` (sets `is_open = true`, resets `selected = 1`). Implement `SettingsMenu:close()` (sets `is_open = false`). Implement `SettingsMenu:update(dt)`: edge-triggered reads of up/down arrows (`love.keyboard.isDown`) to navigate the two items with wrap, and edge-triggered confirm (`e`, `f`, `return`, `space`) to call `_confirm()`. Edge-triggered escape calls `self:close()`. Store `_prev_*` flags. Implement `SettingsMenu:_confirm()`: index 1 calls `love.window.setFullscreen(not love.window.getFullscreen())`; index 2 calls `love.event.quit()`. Implement `SettingsMenu:draw()`: draw a semi-transparent black rectangle over the full 1280×720 logical area (`love.graphics.rectangle("fill", 0, 0, 1280, 720)` at alpha ~0.55), then draw two buttons centred on screen using `start_btn.png` / `start_btn_selected.png` matching the layout constants from `start_scene.lua` (`BTN_W=300`, `BTN_H=54`, `BTN_X=(1280-300)/2`, starting `BTN_Y0` adjusted for two centred items). Label for index 1 is `"Fullscreen"` when windowed (`not love.window.getFullscreen()`) and `"Window"` when fullscreen. Return the module at the bottom.

- [ ] Task 2 — `main.lua` — Wire up `SettingsMenu` in normal game mode. At the top, add `local SettingsMenu = require("lua/game/scenes/settings_menu")`. In `love.load()`, inside the `else` branch (normal mode), after constructing `scene_manager`, add `settings_menu = SettingsMenu.new()` (module-level variable). Pass `function() settings_menu:open() end` as a fourth argument to `StartScene.new(gs, input, scene_manager, <callback>)`. In `love.update(dt)`, inside the `not _visual_mode` branch, check `if settings_menu.is_open then settings_menu:update(dt) else input:update() scene_manager:update(dt) end`. In `love.draw()`, after `love.graphics.draw(canvas, ox, oy, 0, scale, scale)`, add `if settings_menu and settings_menu.is_open then settings_menu:draw() end`. In `love.keypressed(key)`, replace the bare `love.event.quit()` with: if `settings_menu` exists and `scene_manager.current` has `esc_opens_settings == true`, toggle `settings_menu` open/close; otherwise fall through to `love.event.quit()` only when not in a settings-aware scene.

- [ ] Task 3 — `lua/game/scenes/start_scene.lua` — Add `"Settings"` button. Insert `"Settings"` into `ITEMS` at index 3, shifting `"Exit"` to index 4: `local ITEMS = { "New Game", "Continue", "Settings", "Exit" }`. Update `StartScene.new` signature to accept a fourth parameter `open_settings` and store it as `self.open_settings`. In `_confirm()`: index 3 calls `self.open_settings()` and returns; index 4 (formerly 3) calls `love.event.quit()`. The existing `StoreScene` switch for indices 1 and 2 stays unchanged.

- [ ] Task 4 — `lua/game/scenes/store_scene.lua` — Add `self.esc_opens_settings = true` inside `StoreScene.new()`, after the existing field assignments.

- [ ] Task 5 — `lua/game/scenes/buy_scene.lua` — Add `self.esc_opens_settings = true` inside `BuyScene.new()`, after the existing field assignments.
