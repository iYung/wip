# Quest Editor Checklist

Tasks are sequential — each builds on the same file `tools/quest_editor.py`.

- [x] Task A — `tools/quest_editor.py` — Create the file with the Lua parser and writer. The parser reads `lua/game/data/customer_scripts.lua` into a Python list of dicts (all fields: id, chapter, name, trigger, plant_type, messages, after_messages, primary_color, secondary_color, voice_pitch, accessory, no_dismiss). The writer serializes that list back to the same Lua format. Add a `__main__` block that loads the file, round-trips it through the writer, and prints the number of chapters loaded as a smoke test.

- [x] Task B — `tools/quest_editor.py` — Add the tkinter window and two-pane layout. Left pane: a `Canvas` with horizontal and vertical scrollbars (90% of window width). Right pane: a `Frame` placeholder for the edit panel (10%). Bottom bar: `+ Add Quest` button and `Save to Lua` button, both stubs. Window title: "Quest Editor". Size: 1280×720. Wire the window to load `customer_scripts.lua` on open and display a file-not-found error dialog if it is missing.

- [x] Task C — `tools/quest_editor.py` — Implement `draw_canvas()`. Render one horizontal swimlane row per unique character `id`, ordered by the trigger count of their first chapter (ascending). Within each row, draw chapters left-to-right as rounded-rectangle cards connected by arrows. Each card shows: character name + chapter number on line 1, `needs [PlantName] ×N` on line 2, `requests [PlantName]` on line 3. Color each card by its requested `plant_type` using a fixed palette (one distinct color per type 1–6). Draw character id labels on the left margin. Call `draw_canvas()` after loading data on startup.

- [x] Task D — `tools/quest_editor.py` — Add first-appearance badges to `draw_canvas()`. For each plant type (1–6), find the chapter with the lowest trigger count that involves that plant type as either a trigger plant or a requested plant (ties broken by character id alphabetical order). Draw a small gold star and the plant name label in the top-left corner of that card.

- [x] Task E — `tools/quest_editor.py` — Implement the edit panel. When a card on the canvas is clicked, populate the right pane with labeled fields for all editable properties: id (Entry), chapter (Spinbox, int), name (Entry), trigger plant_type (Spinbox 1–6), trigger count (Spinbox ≥ 0), plant_type requested (Spinbox 1–6), messages (Listbox with Add/Remove row buttons and an Entry for new lines), after_messages (same pattern), voice_pitch (Entry, float), accessory (Entry), no_dismiss (Checkbutton), primary_color and secondary_color (buttons showing a colored swatch that open `tkinter.colorchooser`). All field changes update the in-memory data immediately (no separate Apply button). Redraw the canvas after any change.

- [x] Task F — `tools/quest_editor.py` — Wire up Add Quest, Delete Chapter, and Save to Lua. **Add Quest**: clicking the button opens a dialog with a character id field (combobox of existing ids plus free text) and a chapter number (auto-filled to next for existing id); inserts a blank chapter with default values; selects and scrolls to the new card. **Delete Chapter**: a `Delete Chapter` button in the edit panel; shows a confirmation dialog; removes the chapter from the in-memory list; redraws canvas; clears the edit panel. **Save to Lua**: writes the in-memory list back to `lua/game/data/customer_scripts.lua` using the Lua writer from Task A; shows a brief "Saved." status label in the bottom bar; does not quit the tool.
