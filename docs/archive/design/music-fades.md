# Music Fades

## Goal

Add a separate start menu music track that fades out when New or Continue is selected.
Have the background music (background.wav) fade in when the store scene is entered.
Fade duration: 2 seconds for both transitions.

---

## Affected files

**New asset:**
- `assets/music/menu.mp3` — user-provided start menu track (not generated by code; must be added manually before the feature is playable)

**Modified:**
- `lua/game/sound.lua` — manage two named music tracks ("menu" and "bg"), add a per-track fade system and `Sound.update(dt)`
- `main.lua` — call `Sound.update(dt)` each frame
- `lua/game/scenes/start_scene.lua` — play menu music on `on_enter()`; trigger 2s fade-out on New/Continue confirm
- `lua/game/scenes/store_scene.lua` — stop menu music immediately and fade in bg music over 2s on `on_enter()`
- `lua/headless/stubs.lua` — ensure stub source has `play`, `stop`, `isPlaying` no-ops so new Sound calls don't crash tests

---

## What changes

### Two named music tracks in Sound.lua

Replace the single `_music` variable with a table keyed by track name:

```lua
local _music_tracks = {}
-- each entry: { src = <Source>, fade_vol = 1.0, fade_target = 1.0, fade_rate = 0.0, stop_on_done = false }
```

`"menu"` → `assets/music/menu.mp3`
`"bg"`   → `assets/music/background.wav`

`Sound.load()` loads both as looping `"stream"` sources. Menu track plays immediately at `_music_volume`. Bg track is loaded but not played (starts silent and stopped).

### Fade system

Four new public functions:

- `Sound.update(dt)` — called every frame from `main.lua`. Advances all in-progress fades: increments `fade_vol` toward `fade_target` at `fade_rate` per second, clamps, applies `fade_vol * _music_volume` to the source. When fade completes and `stop_on_done` is true, calls `src:stop()`.

- `Sound.play_music(name)` — if source exists: set volume to `_music_volume`, call `src:play()`, clear any in-progress fade. Use this to (re)start a track at full volume instantly.

- `Sound.fade_music(name, target_vol, duration)` — starts a fade. `target_vol` is 0–1 (relative to `_music_volume`). If the track is not playing and `target_vol > 0`, starts playing first (at volume 0 so the fade-in is smooth). If `target_vol == 0`, sets `stop_on_done = true` so the source is stopped when the fade reaches zero. Rate = `(target - current) / duration`.

- `Sound.stop_music(name)` — stops a track immediately, clears its fade state.

### set_music_volume still works

`Sound.set_music_volume(v)` updates `_music_volume`. For each track, if playing, reapply `fade_vol * _music_volume` so volume responds in real time even during a fade.

### StartScene

- `on_enter()`: call `Sound.play_music("menu")` only if menu track is not already playing (guard with `src:isPlaying()`). This handles returning to the start screen from settings without restarting the track from the beginning.
- `_confirm()` for items 1 (New Game) or 2 (Continue): call `Sound.fade_music("menu", 0, 2)` before the `scene_manager:switch(...)` call. The fade continues running in `Sound.update` after the scene switch — no delay needed.

### StoreScene

- `on_enter()`: call `Sound.stop_music("menu")` then `Sound.fade_music("bg", 1, 2)`. The bg track starts at silence and fades to full over 2 seconds.

### main.lua

Add `Sound.update(dt)` call in `love.update(dt)` in the non-visual-mode branch, before `scene_manager:update(dt)`.

## What stays the same

- SFX system (`Sound.play(name)`) is unchanged.
- Settings menu music volume slider still works.
- Headless tests remain silent — stubs handle all new calls as no-ops.
- The fade does not block the scene switch; transitions happen immediately, audio fades in the background.
